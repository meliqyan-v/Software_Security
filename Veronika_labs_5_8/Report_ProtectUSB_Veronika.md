## 1. Նախագծի նպատակը և խնդիրները
Այս աշխատանքի շրջանակներում իրականացվել է USB կրիչների բովանդակության պատճենահանման պաշտպանության համակարգ (**ProtectUSB Content**)։ Նպատակն է ապահովել տվյալների գաղտնիությունը և ամբողջականությունը USB սարքերի վրա՝ կիրառելով գաղտնագրման և մուտքի վերահսկման մեխանիզմներ։

## 2. Տեխնիկական իրականացում և Կոդի վերլուծություն

### Ա. USB Drive Detection (Կրիչների հայտնաբերում)
Համակարգը պետք է ավտոմատ կերպով նույնականացնի միացված USB սարքերը։

**Կոդի հատվածը (C++ տրամաբանություն):**
```cpp
std::vector<std::string> USBDetector::getUSBDrives() {
    DWORD drives = GetLogicalDrives();
    for (int i = 0; i < 26; i++) {
        if (drives & (1 << i)) {
            char rootPath[4] = { char('A' + i), ':', '\\', '\0' };
            if (GetDriveTypeA(rootPath) == DRIVE_REMOVABLE) {
                // Ավելացնել ցուցակին
            }
        }
    }
}
```
**Բացատրություն.** Օգտագործվում է Windows API-ն (`GetLogicalDrives`), որպեսզի սկանավորվեն բոլոր տրամաբանական սկավառակները և առանձնացվեն միայն շարժական (Removable) կրիչները։

### Բ. AES-256 Encryption Engine (Գաղտնագրման շարժիչ)
Տվյալների պաշտպանության համար կիրառվել է ստանդարտ **AES-256** (Advanced Encryption Standard) գաղտնագրման մեթոդը։

**Կոդի հատվածը (JavaScript/CryptoJS):**
```javascript
const encrypted = CryptoJS.AES.encrypt(fileContent, encryptionKey).toString();
```
**Տեխնիկական լուծում.** AES-256-ը հանդիսանում է ժամանակակից ամենահուսալի սիմետրիկ գաղտնագրման ստանդարտը։ Այն ապահովում է տվյալների բարձր մակարդակի պաշտպանություն՝ դիմակայելով նույնիսկ ամենաբարդ հարձակումներին։

### Գ. File Operations & Integrity (Ամբողջականության ապահովում)
Պաշտպանության ռեժիմը միացնելիս համակարգը կատարում է հետևյալ քայլերը.
1.  Կարդում է բոլոր ֆայլերը։
2.  Գաղտնագրում է դրանք և ավելացնում `.aes` ընդլայնումը։
3.  **Ջնջում է բնօրինակ ֆայլերը** (Delete Originals)։

**Կոդի հատվածը:**
```cpp
bool AESCipher::encryptFile(const std::string& input, const std::string& output) {
    // 1. Կարդալ բնօրինակը

    // 3. Պահպանել նոր ֆայլը (.aes)
```

## 3. Անվտանգության վերլուծություն և Գնահատում

### Ա. Ալգորիթմի ընտրություն
Լաբորատոր աշխատանքի շրջանակներում օգտագործվել է **AES-256** (Advanced Encryption Standard) գաղտնագրման մեթոդը։
*   **Առավելություն.** Համաշխարհային ստանդարտ, բարձրագույն անվտանգություն։
*   **Իրականացում.** Օգտագործվել է `crypto-js` գրադարանը՝ ապահովելով 256-բիթանոց բանալիով գաղտնագրում։

### Բ. Security through Obscurity
Համակարգի ինտերֆեյսում ալգորիթմի անվանումը քողարկված է ("Security Access Key"), ինչը անվտանգության լրացուցիչ շերտ է ստեղծում՝ թույլ չտալով հարձակվողին նախապես իմանալ գաղտնագրման մեթոդը։

## 4. Եզրակացություն
Իրականացված համակարգը հաջողությամբ լուծում է USB կրիչների անվտանգության խնդիրը։ Այն պաշտպանում է տվյալները չթույլատրված պատճենահանումից՝ կիրառելով **AES-256** ստանդարտը, որը երաշխավորում է տվյալների անխոցելիությունը։
